# OctoAcme — Role Onboarding Checklist

## Purpose
Provide a standardized checklist for onboarding team members into their roles on OctoAcme projects. Ensures clarity on responsibilities, communication channels, and key processes.

## Universal Onboarding (All Roles)

### Week 1 — Orientation
- [ ] Complete GitHub access and repository setup
- [ ] Review OctoAcme Project Management Overview
- [ ] Review Project Charter / One-pager
- [ ] Understand core principles (customer-first, iterative delivery, psychological safety)
- [ ] Set up communication tools (Slack, email, calendar)
- [ ] Schedule 1:1s with Project Manager and key collaborators
- [ ] Review current project status and roadmap
- [ ] Understand escalation paths and decision-making structure

### Week 2 — Role Deep Dive
- [ ] Review role-specific responsibilities in octoacme-roles-and-personas.md
- [ ] Review octoacme-roles-interaction-matrix.md to understand collaboration points
- [ ] Attend team standup and sprint planning
- [ ] Meet with key collaborators in your interaction matrix
- [ ] Review recent retrospectives and action items

### Week 3 — Process Familiarization
- [ ] Review relevant process docs (Execution & Tracking, Risk Management, etc.)
- [ ] Attend a demo or review session
- [ ] Participate in a backlog refinement or planning session
- [ ] Ask clarifying questions about processes and decisions

### Week 4 — Ramp-Up Complete
- [ ] Confirm understanding of role and responsibilities
- [ ] Identify any gaps in knowledge or process clarity
- [ ] Plan first contribution or project assignment
- [ ] Schedule 30-day check-in with manager

## Developer Onboarding

### Technical Setup
- [ ] Code repository access and branch protections
- [ ] Development environment setup (local, staging, testing)
- [ ] CI/CD pipeline access and documentation
- [ ] Security scanning and linting tools configured
- [ ] Test framework and automation setup
- [ ] Code review and PR workflow training

### Process & Standards
- [ ] Review Definition of Done (DoD)
- [ ] Understand PR expectations (size, description, tests, review)
- [ ] Review code style guide and linting standards
- [ ] Understand test coverage expectations
- [ ] Review branching strategy and naming conventions

### First Assignment
- [ ] Assigned a small, well-scoped story for first sprint
- [ ] Assigned a code review buddy or mentor
- [ ] Complete first PR with guidance and feedback
- [ ] Participate in tech design review or architecture discussion

---

## QA Lead Onboarding

### Testing & Quality
- [ ] Review test plan and testing strategy documentation
- [ ] Understand Definition of Done and acceptance criteria standards
- [ ] Access to test management tools (test cases, defect tracking)
- [ ] Training on QA automation framework and CI integration
- [ ] Review quality metrics and gate criteria

### Process & Collaboration
- [ ] Understand defect triage and severity classification
- [ ] Learn escalation path for blockers and critical issues
- [ ] Review pre-release quality verification checklist
- [ ] Meet with Developers to understand test expectations
- [ ] Meet with Product Manager to discuss acceptance criteria

### First Assignment
- [ ] Assigned to review backlog for testability and acceptance criteria clarity
- [ ] Create test plan for one feature
- [ ] Participate in test case design review
- [ ] Lead first quality gate assessment

---

## UX Designer Onboarding

### Design & Research
- [ ] Access to design tools (Figma, Miro, prototyping software)
- [ ] Review design system and component library
- [ ] Understand design standards and accessibility requirements
- [ ] Review user research methodologies and tools
- [ ] Understanding of design-to-development handoff process

### Process & Collaboration
- [ ] Understand user acceptance testing requirements
- [ ] Review design review and feedback process
- [ ] Learn design documentation standards
- [ ] Meet with Product Manager on user research approach
- [ ] Meet with Developers on implementation feasibility

### First Assignment
- [ ] Participate in user research activity or review session
- [ ] Create design specs or prototypes for a feature
- [ ] Conduct design review with team
- [ ] Incorporate feedback and iterate

## Business Analyst Onboarding

### Requirements & Analysis
- [ ] Review requirements gathering and documentation templates
- [ ] Understand user story and acceptance criteria standards
- [ ] Access to requirement management tools and documentation
- [ ] Training on impact analysis and business rule documentation
- [ ] Review change management and scope control processes

### Process & Collaboration
- [ ] Understand stakeholder communication and approval process
- [ ] Learn clarification and refinement workflows

- [ ] Create or refine user stories and acceptance criteria

## Release Manager Onboarding

### Release & Deployment
- [ ] Review Release & Deployment guide and procedures
- [ ] Access to deployment and release management tools
- [ ] Understand go/no-go criteria and gate assessments
- [ ] Training on rollback procedures and incident response
- [ ] Review release note and documentation templates

### Process & Collaboration
- [ ] Understand release schedule and planning process
- [ ] Learn coordination process across teams
- [ ] Review stakeholder communication and announcement process
- [ ] Meet with DevOps/Infrastructure for deployment procedures
- [ ] Meet with QA Lead on quality verification process

### First Assignment
- [ ] Shadowed on a release cycle
- [ ] Participate in pre-release verification activities
- [ ] Create release notes for a minor release
- [ ] Lead go/no-go assessment with team

## Support Engineer Onboarding

### Support & Monitoring
- [ ] Access to support ticketing and incident management tools
- [ ] Training on post-release monitoring and health checks
- [ ] Understand incident escalation and triage procedures
- [ ] Review customer communication templates and best practices
- [ ] Learn feedback collection and trend analysis tools

### Process & Collaboration
- [ ] Understand escalation path for critical issues
- [ ] Learn bug reporting and defect documentation standards
- [ ] Review hotfix coordination process with Release Manager
- [ ] Meet with Development team on issue investigation
- [ ] Meet with Product Manager on feedback analysis

### First Assignment
- [ ] Shadow support team on post-release monitoring
- [ ] Triage and document a support ticket
- [ ] Escalate and collaborate on issue investigation
- [ ] Collect and summarize customer feedback trends

## Stakeholder Onboarding

### Business Context
- [ ] Review project charter and business objectives
- [ ] Understand success metrics and how they will be measured
- [ ] Review roadmap and strategic alignment
- [ ] Understand decision gates and approval process
- [ ] Review communication cadence and update schedule

### Process & Engagement
- [ ] Understand their role in requirements and approvals
- [ ] Learn feedback and review process for deliverables
- [ ] Review scope change management process
- [ ] Understand escalation and risk communication
- [ ] Schedule recurring status update meetings

### First Engagement
- [ ] Attend project kickoff or milestone review
- [ ] Provide initial feedback on project direction
- [ ] Identify any business constraints or dependencies
- [ ] Confirm alignment on success metrics and timeline

---

## Project Manager Onboarding

### Project Management Setup
- [ ] Access to project management tools (GitHub Projects, etc.)
- [ ] Review project charter and artifacts
- [ ] Understand communication and escalation processes
- [ ] Review risk register and mitigation strategies
- [ ] Access to project budget and resource constraints

### Process & Facilitation
- [ ] Review meeting facilitation best practices
- [ ] Understand decision-making and approval processes
- [ ] Learn reporting cadence and stakeholder communication
- [ ] Review retrospective and continuous improvement process
- [ ] Meet with all team members to understand current status

### First Week
- [ ] Hold team kickoff or alignment meeting
- [ ] Create or review project timeline and milestones
- [ ] Establish communication cadence with team and stakeholders
- [ ] Identify immediate risks and action items
- [ ] Schedule weekly syncs and planning sessions
